# HACS Default Store Submission Checklist

> *Generated by [aurora@aurora-smart-home (ha-integration-dev skill)](https://github.com/tonylofgren/aurora-smart-home)*

Target: get **Open-Meteo Marine Weather** into the **default HACS store** so users
can install it without adding a custom repository.

This checklist covers the steps that happen **outside this project folder** —
on GitHub and in the `home-assistant/brands` and `hacs/default` repositories.
The code in this folder is already shaped for default-store submission
(`version` in manifest, `hacs.json` present, `quality_scale: bronze`).

## 1. Publish the GitHub repository

- [ ] Create/push to `https://github.com/cloudbr34k84/home-assistant-Open-Meteo-Marine-Weather`
      (must match the `documentation` and `issue_tracker` URLs in `manifest.json`).
- [ ] Repository root must contain: `custom_components/`, `hacs.json`, `README.md`,
      `LICENSE`, `.github/workflows/validate.yaml` — i.e. push this folder as-is.
- [ ] Add repository **topics**: `hacs`, `home-assistant`, `homeassistant`,
      `custom-integration`.

## 2. Pass CI

- [ ] The included `.github/workflows/validate.yaml` runs **hassfest** and the
      **HACS action**. Both must be green.
- [ ] A red hassfest run usually means a manifest field is missing or the brand
      (step 3) is not yet registered.

## 3. Register brand assets (home-assistant/brands)

Default HACS requires the domain to exist in the brands repo. The `logo.png`
inside the integration folder does **not** satisfy this.

- [ ] Fork `https://github.com/home-assistant/brands`.
- [ ] Add:
      - `custom_integrations/open_meteo_marine_weather/logo.png`
      - `custom_integrations/open_meteo_marine_weather/icon.png` (square, 256x256+)
      (see the brands repo README for exact size/format rules)
- [ ] Open a PR and wait for merge. Until merged, HACS brand validation fails.

## 4. Cut a release

- [ ] Tag a release, e.g. `v1.0.0`, matching `version` in `manifest.json`.
      HACS installs from GitHub releases, not from the default branch.

## 5. Submit to hacs/default

- [ ] Fork `https://github.com/hacs/default`.
- [ ] Add `cloudbr34k84/home-assistant-Open-Meteo-Marine-Weather` to the `integration` file.
- [ ] Open a PR. The HACS bot validates automatically; fix anything it flags.

## Notes

- **Duplicate-of-core risk:** Home Assistant core ships an official *Open-Meteo
  Weather* integration. This one covers **marine/wave** data that core does not.
  Make that distinction explicit in the repo description and the `hacs/default`
  PR so reviewers don't mistake it for a duplicate.
- **HA Core (built-in) is a different target.** If you ever aim for inclusion in
  `home-assistant/core` instead of the HACS store, you must **remove** the
  `version` field from `manifest.json`, drop `hacs.json`, add a full pytest
  suite, and pass core code review. Do not attempt both tracks with one manifest.
