# Brands submission staging

This folder is **not part of the Home Assistant integration**. It stages the
brand assets required for HACS default-store acceptance, which requires the
integration's icon/logo to be listed in the
[home-assistant/brands](https://github.com/home-assistant/brands) repository.

## How to submit

1. Fork [home-assistant/brands](https://github.com/home-assistant/brands).
2. Copy `custom_integrations/open_meteo_marine_weather/` from this folder into
   the same path in your fork.
3. Open a pull request against `home-assistant/brands` with just that folder
   added — no other changes.
4. Once merged, HACS's brands validation check will pass automatically (it
   checks the brands repo, not this one).

## Assets

Generated from `custom_components/open_meteo_marine_weather/logo.png` (the
existing integration logo), resized to the brands repo's required dimensions:

| File | Size | Purpose |
|------|------|---------|
| `icon.png` | 256×256 | Square icon, standard resolution |
| `icon@2x.png` | 512×512 | Square icon, high-DPI |
| `logo.png` | 206×200 | Landscape logo, standard resolution (shortest side 200px) |
| `logo@2x.png` | 412×400 | Landscape logo, high-DPI (shortest side 400px) |

All are PNG with transparency preserved, generated by padding the source logo
to a square (for the icon variants) and scaling the original aspect ratio
(for the logo variants) — no cropping, so no part of the original artwork
was lost.
